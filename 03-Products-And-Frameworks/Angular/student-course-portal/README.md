# Angular — Student Course Portal

Hands-On **1–10** in a single **Angular 20** application (`student-course-portal`).

## Objective

Progress from components and routing through HTTP/JSON Server, interceptors, RxJS, NgRx state management, and **unit testing** with Jasmine/Karma.

## Folder structure (additions for 8–10)

```
src/app/
├── core/
│   ├── interceptors/     authorization, loading, error (+ *.spec.ts)
│   └── services/         loading.service, http-error.service
├── services/
│   ├── course-api.service.ts   (HttpClient CRUD + RxJS)
│   └── course.service.ts       (facade)
├── store/courses/
│   ├── courses.actions.ts
│   ├── courses.reducer.ts
│   ├── courses.effects.ts
│   ├── courses.selectors.ts
│   └── *.spec.ts               (reducer, selectors, effects)
├── testing/
│   └── test-data.ts            (shared mock courses)
├── features/course-admin/      (NgRx CRUD UI)
├── environments/
db.json                         (JSON Server data)
coverage/                       (generated by ng test --code-coverage)
```

## Hands-On 8 — HTTP & RxJS

| Topic | Implementation |
|-------|----------------|
| **HttpClient** | `CourseApiService` → `http://localhost:3000/courses` |
| **JSON Server** | Root `db.json`, `npm run api` |
| **CRUD** | GET/POST/PUT/DELETE in API service + **Course Admin** UI |
| **RxJS** | `map`, `tap`, `catchError`, `retry`, `switchMap` in `course-api.service.ts` |
| **Interceptors** | Authorization, Loading, Error (functional interceptors) |

## Hands-On 9 — NgRx

| Topic | Implementation |
|-------|----------------|
| **Store** | `provideStore({ courses: coursesReducer })` |
| **Actions** | Load/create/update/delete course actions |
| **Reducers** | `coursesReducer` |
| **Effects** | `CoursesEffects` (HTTP via `switchMap`) |
| **Selectors** | `selectAllCourses`, `selectCoursesLoading`, etc. |
| **Redux DevTools** | `provideStoreDevtools` (enabled in dev mode) |

## Hands-On 10 — Unit testing

| Topic | Implementation |
|-------|----------------|
| **Runner** | Jasmine + Karma (`@angular/build:karma`) |
| **TestBed** | Component, service, pipe, directive, and interceptor specs |
| **HttpTestingController** | `course-api.service.spec.ts` (no real HTTP) |
| **MockStore** | `course-list`, `course-admin` (`provideMockStore`, `resetSelectors` in `afterEach`) |
| **NgRx** | Reducer, selector, and effects specs (`provideMockActions` for effects) |
| **Coverage** | `codeCoverage: true` in `angular.json`; report under `coverage/student-course-portal/` |

**Run tests (watch mode):**

```bash
ng test
```

**CI / headless + coverage:**

```bash
npm run test:ci
```

Open `coverage/student-course-portal/index.html` for the HTML coverage report.

Spec files live next to the code they exercise (`*.spec.ts`) plus `src/app/testing/test-data.ts`.

## How to run

**Terminal 1 — JSON Server (required for live data):**

```bash
cd 03-Products-And-Frameworks/Angular/student-course-portal
npm run api
```

**Terminal 2 — Angular app:**

```bash
npm install
npm start
```

Open `http://localhost:4200/`  
Redux DevTools browser extension shows NgRx actions/state.

**Production build (no API required for compile):**

```bash
ng build
```

## Key routes

| Route | Description |
|-------|-------------|
| `/courses` | Catalog (dispatches `loadCourses`, reads store) |
| `/course-admin` | CRUD + NgRx (Auth Guard) |
| `/login` | Sets Bearer token for Authorization interceptor |

## Expected output

- Global **Loading…** bar during HTTP calls (Loading interceptor).
- **Authorization** header `Bearer demo-token-<username>` after login.
- HTTP failures show dismissible **global error** banner (Error interceptor).
- **Course Admin**: create/edit/delete updates JSON Server and NgRx store.
- DevTools: ` [Courses] Load Courses`, success/failure, CRUD actions.
- **Unit tests**: `49` specs, all passing; coverage summary printed after `npm run test:ci`.
